#  Текущая оценка и цели проекта drv60870

**Дата:** 2025-03-14

## Цели проекта

1. **Симулятор полевого уровня телемеханики** — имитация КП (контроллеров/РТУ) по протоколу МЭК 60870-5-104 для проверки SCADA (WinCC OA и аналоги).
2. **Учебный характер** — получение инструментов тестирования, изучение протокола 104 и Python.

---

## Текущее состояние

### Реализовано

| Компонент | Статус | Комментарий |
|-----------|--------|-------------|
| Сервер (listen, accept) | ✅ | Порт из config.toml, фильтр по `allow_ip` |
| U-кадры | ✅ | STARTDT/STOPDT/TESTFR — приём и ответ |
| Приём I-кадров | ✅ | Парсинг, обновление N(R) (rec_sq) |
| C_IC_NA_1 (общий опрос) | ✅ | ACT_CON, выгрузка всех сигналов (COT=INTERROGATED), ACT_TERM |
| C_SC_NA_1 (однопозиционная ТУ) | ✅ | ACT_CON, значение пишется в хранилище (on_command) |
| Отправка I-кадров | ✅ | M_SP_NA_1(1), M_SP_TB_1(30), M_ME_NC_1(13), M_ME_TF_1(36) |
| Хранилище сигналов | ✅ | DataStorage с подпиской клиентов, потокобезопасно |
| Конфиг | ✅ | TOML (сеть, протокол, лог), CSV сигналов (id, ca, ioa, asdu, name, val) |
| Имитация | ⚠️ Минимально | `imit.sim_float` + консольная команда `imit N id` |
| S-кадры | ✅ | Приём: парсинг N(R) (handle_s_frame). Отправка: по параметру w (build_s_frame, rec_count_since_send) |
| Клиент 104 | ❌ | В репозитории — тестовый TCP-клиент (порт 3051), не протокол 104 |

### Пробелы для роли «симулятор для SCADA»

- **Сценарии имитации** — воспроизводимые проверки SCADA требуют: сценарий по времени из CSV (время, id, значение) и/или циклическая имитация по группе сигналов с интервалом.
- **Документация и учебная ценность** — краткое описание 104 в контексте проекта, как подключать WinCC OA к симулятору, как запускать сценарии для проверок.

### Учебная ценность (уже есть / можно усилить)

- Изучение 104: разбор кадров в `protocol.py`, константы в `const.py`, формат CP56 в `protocol.py`.
- Python: threading, queue, dataclasses, конфиг (tomllib, csv).
- Инструмент тестирования: несколько экземпляров КП (KP_1, KP_2), один сервер на инстанс, сценарии для регрессий.

## Рекомендуемые приоритеты

1. **Имитация** — сценарий из CSV (время, id, значение); циклическая имитация по группе/интервалу.
2. **Документация** — цель проекта, структура, как тестировать SCADA (WinCC OA), примеры сценариев (в README или `docs/`).
3. **Опционально** — телерегулирование (C_SE_NC_1/C_SE_TC_1), двухпозиционная команда (C_DC_NA_1); клиент 104 для обратной проверки.

## Итог

Проект пригоден как симулятор и учебный стенд: общий опрос (C_IC_NA_1), запись ТУ (C_SC_NA_1) и S-кадры (приём N(R), отправка по w) реализованы. Сценарии имитации и краткая документация доведут роль «симулятор полевого уровня + учебный проект» до целевого состояния.

## Обновления реализации (после 2025-03-14)

- **Конфиг:** `send_sleep`, `log_i_frame_stats_every` (prot/log в TOML).
- **Хранилище:** индекс по IOA (`_ioa_idx`), `update_val(new_val, id=…|ioa=…)`, `get_all_for_gi()` для общего опроса; у сигнала поле `threshold` (в CSV), рассылка только при превышении порога.
- **Протокол:** общий опрос выгружает события через `on_gi`, в конец — ACT_TERM (asdu 100, cot 10); C_SC_NA_1 вызывает `on_command(val, ioa)`; приём S-кадров обновляет `last_ack_nr`; отправка S по w (`rec_count_since_send`).
- **Сервер:** ограничение по k (не более k неподтверждённых I-кадров), TCP_NODELAY; реестр CLI-команд (set, setioa, imit, help и др.).
